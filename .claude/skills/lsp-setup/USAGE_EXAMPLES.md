## Basic Usage

### Example 1: New Python Project

**Scenario**: Starting a new Python project with virtual environment

```bash
$ mkdir my-python-app && cd my-python-app
$ python -m venv .venv
$ source .venv/bin/activate
$ pip install django pytest

# Create some Python files
$ cat > main.py << EOF
from django.http import HttpResponse

def hello(request):
    return HttpResponse("Hello, world!")
EOF

$ cat > test_main.py << EOF
def test_hello():
    assert True
EOF

# Configure LSP
$ /lsp-setup

[LSP Setup] Detecting languages...
✓ Found: Python (2 files)
  - Detected framework: Django
  - Virtual environment: .venv

[LSP Setup] Configuring LSP servers...
✓ pyright: Installed at /usr/local/bin/pyright

[LSP Setup] Configuring project...
✓ Created .env with LSP configuration:
  LSP_PYTHON_INTERPRETER=/path/to/.venv/bin/python
  LSP_PYRIGHT_PATH=/usr/local/bin/pyright
✓ Configured pyright to use Django settings

[LSP Setup] Verifying connections...
✓ pyright: Connected (Python 3.11.5)
  - Django support: Enabled
  - Type checking: Basic mode

Configuration complete! LSP servers ready.

# Verify it works
$ cat .env
# LSP Configuration - Auto-generated by /lsp-setup on 2024-12-01 14:23:15
ENABLE_LSP_TOOL=1
LSP_PYTHON_INTERPRETER=/path/to/my-python-app/.venv/bin/python
LSP_PYRIGHT_PATH=/usr/local/bin/pyright
LSP_PYTHON_FRAMEWORKS=django
```

**Result**: Python LSP server configured with Django support and correct virtual environment

## Advanced Workflows

### Example 15: CI/CD Integration

**Scenario**: Verify LSP configuration in CI pipeline

```bash
# In .github/workflows/lsp-check.yml
name: LSP Configuration Check

on: [push, pull_request]

jobs:
  lsp-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Install Claude Code
        run: npm install -g @anthropic-ai/claude-code
      - name: Install LSP servers
        run: |
          npm install -g pyright vtsls
      - name: Verify LSP configuration
        run: /lsp-setup --status-only
      - name: Check configuration file
        run: |
          if [ ! -f .env ]; then
            echo "Error: .env not found"
            exit 1
          fi
          if ! grep -q "LSP_" .env; then
            echo "Error: No LSP configuration in .env"
            exit 1
          fi
          echo "LSP configuration valid"
```

**Result**: Automated LSP configuration validation in CI
